Evaluation Criteria

Skill 评价标准 v2

基于 Anthropic 官方规范、Agent Skills 标准、Vercel skills CLI 文档及多个 review/benchmark 项目总结。采用「三层漏斗」：Registry 初筛 → Review 静态审查 → Benchmark/实测。

维度权重为什么重要评价依据 / 来源
1. 规范符合度 20%一票否决 没有 SKILL.md 和 frontmatter，agent 无法识别和加载该 skill。 Anthropic 官方 README："Skills are folders... with a SKILL.md file... frontmatter requires name and description."
📌 一手来源
2. Agent 兼容性 15%一票否决 skill 再好，不支持你用的 agent 就无意义。 Vercel skills CLI README 列出支持 70+ agent；agentskills.io 定义 discovery/activation/execution 三阶段加载机制。
📌 一手来源
3. 功能描述准确完整 20% 决定 agent 能否在正确时机激活该 skill，以及用户能否预判其能力边界。 Anthropic 官方 skill 模板要求 description 必须说明 "what this skill does and when to use it"。
📌 一手来源
4. 实用示例与边界说明 15% 示例决定用户能否快速上手；边界说明避免误用和幻觉。 Anthropic 官方模板包含 `## Examples` 和 `## Guidelines`；Terryc21/skill-reviewer 的 discoverability lens 专门检查激活词是否清晰。
📌 一手 + 📚 二手
5. 文档结构清晰度 10% 影响可维护性和团队协作。结构混乱的 skill 难以复用。 来自 skill-reviewer 的 architecture lens 和 parseability lens。
📚 二手来源
6. 活跃度 10%一票否决 AI 工具链、模型 API、agent 协议变化极快，长期不更新的 skill 可能失效。 GitHub `pushedAt` / `updatedAt`；Anthropic README 免责声明强调 "Always test skills thoroughly"。
📌 一手来源
7. 可信与安全 10%一票否决 skill 可执行脚本、读写文件、调用网络，来源不明可能带来数据泄露或破坏。 skill-reviewer 的 safety lens 检查备份、恢复、敏感操作；许可证信息来自 GitHub API。
📌 一手 + 📚 二手

评分公式

静态质量分 = 规范×0.20 + 兼容×0.15 + 功能描述×0.20 + 示例边界×0.15 + 文档×0.10 + 活跃×0.10 + 安全×0.10

重要：这 7 个维度只回答「这个 skill 是否值得试用」。真正的「好不好用」必须在真实任务上测试（Benchmark/实测层）。
